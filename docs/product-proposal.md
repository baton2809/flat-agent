# Product Proposal: FlatAgent

## Обоснование идеи

Покупка квартиры в России - сложный, многоэтапный процесс с высокой когнитивной нагрузкой. Ключевые проблемы:

- **Расчёт ипотеки:** банки дают разные цифры, ставки меняются вслед за ключевой ставкой ЦБ, пользователь не знает реальной переплаты
- **Сравнение вариантов:** первичка vs вторичка требует учёта разных рисков (ДДУ, эскроу, состояние дома) и источников данных
- **Разрыв контекста:** каждый визит в банк или агентство начинается с нуля, агент помнит предпочтения пользователя

FlatAgent закрывает все эти точки в одном интерфейсе — Telegram-боте с подключённым AI-агентом. Агент использует три источника внешних знаний: актуальные ставки ЦБ РФ (API), веб-поиск по рынку (DuckDuckGo), и пользовательские данные о конкретных объектах (прайс-листы застройщиков в формате CSV с последующим ML-анализом).

## Цель и метрики

**Цель:** создать работающий PoC AI-агента для консультаций по недвижимости, который демонстрирует применимость LangGraph + GigaChat для задач с многошаговым рассуждением и внешними инструментами.

### Продуктовые метрики

| Метрика | Цель PoC |
|---|---|
| Корректность расчёта ипотеки | Расхождение с банковским калькулятором <= 1% |
| Точность роутинга запросов | >= 90% на тестовом наборе из 50 запросов |
| Успешность поиска | Релевантные результаты в >= 80% поисковых запросов |

### Агентские метрики

| Метрика | Цель PoC |
|---|---|
| Точность извлечения памяти пользователя | >= 85% (имя, бюджет, предпочтения сохраняются корректно) |
| Количество шагов до ответа | <= 3 шага для стандартных сценариев |
| Процент fallback на общий chat | <= 15% от запросов, которые должны идти в specialised node |

### Технические метрики

| Метрика | Цель PoC |
|---|---|
| p95 latency (простой вопрос) | <= 5 секунд |
| p95 latency (поиск + суммаризация) | <= 15 секунд |
| Uptime API | >= 99% в период демо |
| Размер checkpoint в SQLite | <= 100 KB на пользователя |

## Потенциальные сценарии использования

### Основные сценарии

1. **Расчёт ипотеки**
   - "Посчитай ипотеку 6 млн на 20 лет"
   - "Сколько переплачу если взять на 15 лет вместо 20?"
   - "Какая сейчас ключевая ставка?"

2. **Сравнение вариантов**
   - "Что лучше: новостройка за 9 млн или вторичка за 7.5 млн в Москве?"
   - "Сравни риски ДДУ и готового жилья"

3. **Поиск по рынку**
   - "Найди однушки в Митино до 8 млн"
   - "Что сейчас происходит с ценами на недвижимость в Москве?"

4. **Анализ прайс-листа застройщика**
   - Пользователь загружает спарсенный прайс-лист с сайта застройщика (CSV)
   - Агент автоматически определяет структуру колонок, строит OLS-регрессию цена/площадь, выявляет выбросы, выдаёт график с трендом и аналитический вывод: какие лоты переоценены, какие — ниже рынка

5. **Консультации**
   - "Что такое эскроу-счёт?"
   - "Какие документы нужны для сделки?"
   - "Можно ли использовать маткапитал при ипотеке?"

### Edge-кейсы

| Сценарий | Ожидаемое поведение |
|---|---|
| Запрос без суммы ипотеки | Агент запрашивает недостающие параметры |
| CSV с нестандартными колонками застройщика | Агент пытается угадать маппинг через LLM, при неудаче просит пользователя уточнить названия колонок |
| Поиск по нерелевантной теме | Роутер направляет в chat-node, GigaChat отвечает в рамках компетенции |
| Очень длинная история диалога | SqliteSaver сохраняет, но передаётся только sliding window последних N сообщений |
| GigaChat недоступен | FastAPI возвращает 503 с понятным сообщением об ошибке |
| ЦБ API недоступен | Агент использует последнее известное значение ставки с пометкой о давности данных |
| Одновременные запросы одного пользователя | Последовательная обработка через thread_id lock в LangGraph |
| Пользователь пишет на английском | GigaChat отвечает на том же языке |

## Ограничения

### Технические (SLO)

| Параметр | Значение |
|---|---|
| p95 latency (chat) | 5 сек |
| p95 latency (search) | 15 сек |
| p99 latency | 30 сек |
| Макс. длина контекста | 32K токенов (GigaChat Pro) |
| Макс. размер CSV | 5 MB |
| Одновременных пользователей | 10 (PoC, single instance) |

### Операционные

| Параметр | Значение |
|---|---|
| Бюджет на GigaChat API | ~2000 руб/месяц (PoC-трафик) |
| Инфраструктура | Single VPS, 2 CPU / 4 GB RAM |
| Без авто-масштабирования | PoC не требует |
| Telegram webhook | Требует публичный IP или ngrok |

## Архитектурный набросок

```
Telegram Bot / FastAPI
        |
        v
   LangGraph StateGraph
        |
   [router_node]  <-- GigaChat function calling
        |
   +----+----+----+
   |    |    |    |
mortgage compare search chat
   |    |    |    |
   +----+----+----+
        |
   [memory_extraction_node]  <-- сохранение фактов о пользователе
        |
      SQLite
   (checkpoints + user_memory)
```

**Модули:**
- `agent/graph.py` - сборка LangGraph StateGraph
- `agent/nodes/` - 6 узлов: router, mortgage, compare, search, chat, memory_extraction
- `agent/tools/` - инструменты: cbr_tool (ЦБ РФ), search_tool (DuckDuckGo), mortgage_calc, csv_analysis
- `agent/memory.py` - долгосрочная память пользователя (SQLite)
- `agent/llm_wrapper.py` - обёртка над GigaChat (BaseChatModel)
- `api/routes.py` - FastAPI endpoints
- `telegram_bot/bot.py` - Telegram webhook

**Интеграции:**
- GigaChat API (langchain-gigachat)
- ЦБ РФ XML API (ключевая ставка, курсы валют)
- DuckDuckGo Search (без ключа)
- Telegram Bot API

## Data Flow и делегирование LLM/Agent

```
Пользователь -> сообщение
    |
    v
router_node
    |-- [LLM] GigaChat анализирует намерение, выбирает маршрут
    |-- fast-path: regex/keywords для очевидных случаев (ипотека, сравни)
    |
    v (в зависимости от маршрута)
mortgage_node
    |-- [Детерминировано] Парсинг суммы, срока, взноса из текста (regex)
    |-- [Tool] cbr_tool -> ЦБ РФ API -> актуальная ставка
    |-- [Детерминировано] Аннуитетная формула -> точный расчёт
    |-- [LLM] GigaChat форматирует ответ в читаемый вид

search_node
    |-- [LLM] GigaChat формулирует поисковый запрос
    |-- [Tool] DuckDuckGo -> top-5 результатов
    |-- [LLM] GigaChat суммаризует результаты для пользователя

compare_node
    |-- [LLM] GigaChat извлекает параметры для сравнения
    |-- [Tool] cbr_tool -> актуальные ставки
    |-- [LLM] GigaChat генерирует структурированное сравнение

memory_extraction_node (все маршруты)
    |-- [LLM] GigaChat извлекает факты о пользователе из диалога
    |-- [Детерминировано] Сохранение в SQLite user_memory

Что делегировано LLM:
- Классификация намерения (routing)
- Формулировка поисковых запросов
- Извлечение фактов и параметров из естественного языка
- Генерация финального ответа пользователю
- Структурирование сравнений

Что НЕ делегировано LLM (детерминированная логика):
- Расчёт аннуитета (формула)
- HTTP-запросы к ЦБ РФ
- Парсинг CSV прайс-листа застройщика и OLS-регрессия (цена/площадь, выявление выбросов)
- Сохранение/загрузка из SQLite
- Валидация входных параметров (сумма > 0, срок в годах)
```
